    About git study

    基础的上传方式

    1. 先建立一个空文件夹code来clone GitHub博客

    2. clone后的结果是所有GitHub上内容都显现出来，然后复制要上传的文件在code文件中

    3. git status命令可以查看文件处于工作区（红色），暂存区（绿色），历史区（显示无操作）

      • git add.(-A)对文件进行工作区到暂存区转移，git commit -m“解释说明” 对文件暂存区向历史区上传
      • git reset+地址可以恢复之前的操作版本内容
      • git log是对所有的操作版本进行查看
      • git diff< id1 >< id2 >比较当前两个文档之间的区别

    4. 最后git push命令使得历史区push到GitHub上面

    遇到的问题以及解决过程

    1. clone地址的问题

      • 关于clone链接问题:在一次clone地址选取的是HTTP链接，存在一个merge冲突问题，在第二次再次上传过程中使用SSH链接就解决了这个问题
        [merge冲突操作]： D:\假期学习\截图&切图\217.png “merge冲突操作”
        ![merge冲突操作]
      • 代码在GitHub上面成功上传后但在网页上无法显示（考虑到是否没有将项目放到博客上去）
      • 注：昨天使用的方法是在网页中显示了但无法上传到GitHub上

    2. 经过请教学习了解到可以通过一下方法实现上传

      • 打包成一个文件夹放在post新建文件中（存在解决渲染的问题）

      • skip_render:+文件名称避免渲染

      • 然后在post下面git bash here上传就好啦
